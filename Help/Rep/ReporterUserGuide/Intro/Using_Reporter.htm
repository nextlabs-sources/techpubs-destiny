Using Reporter

Any user with permission to log into Reporter can create new reports, on the My Reports tab. After a user creates and runs a report, he has the option of either discarding it or saving it for future reuse. Once saved, reports display in the list on the upper left pane. The user who created a report is considered its owner.

About Shared Reports

For all saved reports, the owner has the option of either sharing them with other users or not sharing them.

The upper left pane on the My Reports tab displays a list of all previously saved reports available to the currently logged in user. This list includes two kinds of reports:

·        All reports owned by that user, whether they are shared or not

·        All reports owned by other users and designated as shared

Note that shared reports may be used by any user who logs into Reporter—that is, any user can run them to refresh the data defined by that query. However, even if reports are shared, other users cannot change any aspects of their defi­nition; only the report owner can do that.

The Basic Workflow

The following is a brief summary of the basic workflow of using Reporter, for both creating and running new reports, and running saved reports. The letter references are to the callouts in the accompanying figures. For complete details on creating and using reports, refer to Using Reports.

Creating a New Report

  1. Log into Reporter.

  2. Click the New button to open a new blank report in the Reports pane.

  3. In the Show field, select the desired report format from among the four choices.

  4. Using the From and To fields, define the reporting period you want the report to cover.

    Defining Reports

  5. Define the report query:

    6. o                   Select the report type, either Policy Activity or Document Activity.

    o                   In the User field, specify some users and/or user components to report on, if desired; otherwise leave the default, Any User. You can use the Lookup button to browse for defined users and components.

    o                   In the Resource Path field, specify some data resources to report on, if desired; otherwise leave the default, Any Resource.

    o                   Policy Activity Reports only: In the Policy field, specify one or more policies to report on, if desired; otherwise leave the default, Any Policy. You can use the Lookup button to browse through a list of defined policies.

    o                   In the Event Level field, specify the level of events you want included in the report, or leave the default, User Events.

    o                   In the Action field, select one or more user actions you want included in the report. If you do not select any, the report will include all actions.

    o                   Policy Activity Reports only: In the Enforcements field, specify the enforcement result you want to filter the report by: Deny, Allow/Monitor, or Both.

  6. Click the Run button to run the report. The results will display in the main Reports pane, and a summary of details about the report’s definition will display in the Selected Report Details pane, in the lower left corner of the window.

  7. If you want to make further changes in the query definition, click the Edit button in the Selected Report Details pane.

  8. If you want to save the report for reuse later, provide values for the Name and Description fields in the Selected Report Details pane, and click the Save button.

  9. If you want to share the report so that other users can run it, check the Is Shared box in the Selected Report Details pane before you save the report.

Running a Saved Report

  1. Log into Reporter.

  2. In the Saved Reports pane, select the report you want to run, to open it in the Reports pane.

  3. If you like, you can change the settings for the Report Type and the To and From reporting period.

  4. If you are the report’s owner, you can optionally change one or more aspects of the query definition before running it. If the report was shared by another owner, you cannot redefine the query, but can only run it as is.

  5. Click the Run button to run the report against the current data in the Activity Journal. The results will display in the main Reports pane, and a summary of details about the report’s definition will display in the Selected Report Details pane.
