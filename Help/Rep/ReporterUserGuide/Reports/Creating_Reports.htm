Creating Reports

In Control Center, a report is a formatted view of a particular set of information extracted from the Activity Journal. You define a report by specifying the query definition—that is, one or more filters for the data contents you want in the report—and the runtime report settings, which determine how the report will actually run. The following table shows the available options for both catego­ries.

Note: Reports may not display properly in Internet Explorer 9 unless the browser is set to compatibility mode.

Setting

Description

Query Criteria

Search

Provides two options: Policy Activity, or Document Activity.

For Policy Activity, the report will show all qualified instances of policies being enforced, consistent with the other query criteria.

For Document Activity, the report will show all activity in the system, consistent with the other query criteria—whether any policies were involved or not.

User

Specifies one or more users whose activity this report will show. Click on the Lookup button to open the search window, where you can select from all currently defined users.

Action

Specifies one or more actions on which to filter this report. The scroll list shows all currently defined actions; press Ctrl to select more than one from the list.

Note that the actions available to filter by are different for Policy Activity and Document Activity reports.

Also, because policies involving Paste actions do not support logging obligations, instances of their enforcement will not be included in any reports.

Resource

Specifies a resource path on which to filter this report. Note that you can filter only by location (path)-based resources, not by file property-based ones.

Policy

Specifies one or more policies on which to filter this report. Click on the Lookup button to open the search window, where you can select from all currently defined policies.

Enforcements

Allows you to filter on enforcements: Allow\Monitor, Deny, or both.

Event Level

Allows you to select the level of event verbosity your report will contain:

  • User-level events only (default)

  • Application- and user-level events

  • All system, application, and user-level events

As a rule, you should leave this setting at the default, User Events only.

Runtime Report Settings

Show

Specifies the report display format: either an Event Details (tabular grid) or one of the grouped bar charts as described in Available Report “Show” Formats.

From

Defines the start date of the interval this report will cover. Click on the calendar icon to choose a date, or type it manually.

To

Defines the end dates of the interval this report will cover. Click on the calendar icon to choose a date, or type it manually.

When you run a report, Reporter compares the information currently in the Activity Journal to the report criteria, selects the matching information (if any), calculates the totals, and displays the results in the format you have selected in the Show field.

Once you define and save a report, it is listed in the Saved Reports pane. The report’s creator can modify or delete the definition of the report; if the report is shared, others can run it and view the results, but cannot modify or delete it.

Policy Activity vs. Document Activity

Whenever you create a new report, you must designate it as one of two types:

·        Policy Activity Report: Provides information that is directly related to enforcement of policies. For example, you could generate a report list­ing all users who tried to open documents that are restricted by a cur­rently deployed policy.

·        Document Activity Report: Provides information that is automatically recorded in the Activity Journal but is not necessarily related to policy enforcement. For example, you can generate a report of all cases when someone stopped a Desktop Enforcer, or read its log files; or when users logged in and out. The exact type of information available depends on the Document Activity Audit Level of the relevant enforcer profile(s).

Depending on which type you select, a different set of query fields will display, as shown in the following figure. There are two key differences between these two interfaces:

·        Policy Activity reports allow you to specify the name of the policy or pol­icies you are interested in, and the enforcement results; while docu­ment activity reports do not.

·        The two types present a different set of Actions on which to filter your query. For details, see Filtering by Action.

Defining a New Report

To define a new report, go to the My Reports tab and click the New button, to display the report’s details in the main Report pane. All fields are open for edit­ing.

 

When you are creating a a new report, you define both the runtime settings and the query definition. For the runtime settings,

  1. In the Show field, select the output format you want for the report. Avail­able Report “Show” Formats provides information on all available formats; for more details and examples, see Working with Reports.

  2. In the To and From fields, define the time period of the user activity you want the report to reflect. You can click on the calendar icon to open the date picker tool, or type the values directly, in a MM DD YYYY format.

Running an Existing Report

You can run any report listed in the Saved Reports. As we mentioned earlier, that includes all of your own reports, and any reports that other report owners have designated as shared.

To run an existing report,

  1. Select the name of the report you want to run. The definition of that report appear in the main Report pane.

    2. If you own the report, all query definition and the runtime setting fields are open for editing. You can change any of the settings, if you wish, or run the report as is.

    If the report is shared by another owner, the runtime settings are open for editing, but the query definition fields are disabled, and must be used as is.

  2. For the Show setting, either leave the default value, or select another output format if you prefer.

  3. For the to and From setting, specify a start date and an end date to define the report period (see below).

  4. Click Run Report. When Reporter finishes generating the report, it displays the results in the format you selected.

  5. If you own the report, you can also click the Edit button to go back to edit mode in the Report pane, to adjust the report’s definition before saving.

About the Report Period

The report period refers to the span in which the data about the activity you were interested in was logged in the Activity Journal. The Report Server only saves data in that set of tables for a limited period of time before moving it to a different set of Archive tables. It is possibly to define a query that includes data that has already been archived—in other words, a query whose report period begins at some time earlier than the oldest data stored in the Activity Journal. In such cases, Reporter will display an error message displaying the date of the earliest available data. The report will generate, but will reflect only the cur­rently available data. You then have the option of using the report as is; redefin­ing the report period (setting a later start date) and rerunning it; or unarchiving the data in question so that it is once again available in the Activity Journal, and then rerunning the report.

Note: For more details on Report Server’s data management features, includ­ing archiving and unarchiving, refer to the “Database Management” chapter of the Control Center Administrator’s Guide.
