Is the deployment time set for Now?

A common mistake that is easily caught, is to fail to set the deployment time to Now. This is controlled by the radio buttons at the top of the deployment screen, as shown below. The default setting is the top button, which is midnight of the current day; if you don’t change that, you may think you have deployed your policy when in fact it is still in Pending Deployment status.

To check this, open the policy in Policy Author and redeploy it, making sure you click the bottom radio button as shown. The time for this setting is editable; by default it is the current time.
