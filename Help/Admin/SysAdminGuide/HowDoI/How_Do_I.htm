How do I . . .

Procedure

Enroll users and user groups?

Use the Enrollment Manager utility, with Type = DIR (for LDAP directories) or Type = LDAP (for LDAP output files). From LDAP directory, requires a connection file and a definition file; from LDIF file, requires only a definition file. See How to Enroll Users and Hosts.

Enroll hosts and host groups?

Use the Enrollment Manager utility, with Type = DIR (for LDAP directories) or Type = LDAP (for LDAP output files). From LDAP directory, requires a connection file and a definition file; from LDIF file, requires only a definition file. See How to Enroll Users and Hosts.

Enroll applications?

Use the AppDiscovery utility, as described on Enrolling Applications.

Enroll Windows file shares?

Use the Resource Path Discovery utility, as described on Enrolling Windows Shares.

Enroll Linux file shares?

Enroll Windows file shares, then use the Samba Directory Mapping utility as described on Linux File Servers.

Enroll location sites?

Use the Enrollment Manager utility with Type = TEXT. Requires a definition file containing a list of one or more sites, expressed as ranges of IP addresses. See Enrolling Location Sites.

Enroll SharePoint sites?

Use the Enrollment Manager utility with Type = PORTAL. Requires a connection file and a definition file, as described on Enrolling SharePoint Groups.

Although it seems as though you enroll SharePoint sites, technically you just enroll SharePoint groups, and register a connection to the SP server that can be used whenever you need browse the site content while defining portal components.

 

Note: Detailed instructions for enrolling all kinds of entities are provided in Enrolling Users, Hosts, and Groups and Enrolling Other Entities of this manual.

How do I . . .

Procedure

Stop a Desktop Enforcer?

1. In Administrator, find which profile the enforcer is using: Status tab, Policy Enforcer Status link: set Show filter to All Desktop Enforcers, check Profile Name column.

2. Find out the administrative password for that profile. (These do not display anywhere; they must be stored outside of Control Center. The default is password.)

3. Use the StopDesktopEnforcer.exe utility, at C:\Program Files\NextLabs\Desktop Enforcer\Policy Controller\public_bin (profile password is required). For details, see the Enforcer Administrator’s Guide.

Start a Desktop Enforcer?

Runs as a Windows service. Manually restart the service under Control Panel, Administrative Tools, Services; or restart the host PC. For details, see the Enforcer Administrator’s Guide.

Stop a Desktop Enforcer from displaying enforcement messages?

Right-click on the EDP icon in the system tray, and uncheck Display Notifications.

Stop a File Server Enforcer?

1. In Administrator, find which profile the enforcer is using: Status tab, Policy Enforcer Status link: set Show filter to All File Server Enforcers, check Profile Name column.

2. Find out the administrative password for that profile. (These do not display anywhere; they must be stored outside of Control Center. The default is password.)

3. Windows: Use the StopFileServerEnforcer.exe utility, under Start, All Programs (profile password is required).

Linux: Log in as local administrator and use the cefse_stop command.

For details, refer to the user’s guide for the respective enforcer.

Restart a File Server Enforcer?

Windows: Runs as a Windows service. Manually restart the service under Control Panel, Administrative Tools, Services; or restart the host computer.

Linux: Log in as local administrator and use the service cefse start command.

For details, refer to the user’s guide for the respective enforcer.

Stop a SharePoint Enforcer?

1. In Administrator, find which profile the enforcer is using: Status tab, Policy Enforcer Status link: set Show filter to All Desktop Enforcers, check Profile Name column.

2. Find out the administrative password for that profile. (These do not display anywhere; they must be stored outside of Control Center. The default is password.)

3. Use the StopSharePointEnforcer.exe utility, under Start, All Programs (profile password is required).For details, refer to the SharePoint Enforcer User’s Guide.

Restart a SharePoint Enforcer?

Runs as a Windows service. Manually restart the service under Control Panel, Administrative Tools, Services; or restart the host PC. For details, refer to the SharePoint Enforcer User’s Guide.

Change the security password for any enforcer?

1. In Administrator, find which profile the enforcer is using: Status tab, Policy Enforcer Status link: set Show filter to All Desktop Enforcers, check Profile Name column.

2. On the Policy Enforcer Configuration tab, select that profile, and provide a new password in the Administrative Password and Confirm Password fields (lower right). See Modifying a Policy Enforcer Profile.

Point an enforcer to a different ICENet Server after it has been installed?

Associate the enforcer host to a different enforcer profile, which is associated with the new ICENet Server. See Moving Profiles between ICENet Servers.

Change the default To and From field contents for e-mail notifications?

These are stored as elements in the <MessageHandlers> section of the config file. See Message Handler Settings.

  

How do I . . .

In Administrator:

Check how many policies are deployed in my network?

Status tab, Status Overview link, Policies statistic (lower left).

Find out what hosts my Control Center components are installed on?

Status tab, Status Overview link, Server Status pane (at right): Host column.

Check the health of all my Control Center server components?

Status tab, Status Overview link, Server Status pane at right: Last Heartbeat value turns red if expected interval has been exceeded.

Check how many enforcers are currently running?

Status tab, Status Overview link, Policy Enforcers statistics (middle left): shows File Server Enforcers, Desktop Enforcers, and Total.

Find out which enforcers are currently running?

Status tab, Policy Enforcer Status link: set Show filter to All Policy Enforcers.

Check if any enforcers are disconnected?

Status tab, Status Overview link, Policies Statistics (lower left). See Status Overview.

Find out which enforcers are disconnected?

Status tab, Policy Enforcer Status link: set Show filter to All Policy Enforcers with Warnings; then check the leftmost column for enforcers with Warning icons (exclamation points). See Status Overview.

Check for enforcers that are using obsolete policies?

Status tab, Status Overview link, Policy Consistency statistic (upper left). See Status Overview.

Find out which enforcers are using obsolete policies?

Status tab, Policy Enforcer Status link: set Show filter to Policy Enforcers with Warnings, look for red X icon in Policy Up-to-Date column. See Policy Enforcer Status.

Tell how often policies are being enforced?

Status tab, Status Overview link, Policies statistic (lower left). See Status Overview.

Find out which profile an enforcer is using?

Status tab, Policy Enforcer Status link: set Show filter to All Policy Enforcers, check Profile Name column. See Policy Enforcer Status.

Find out current heartbeat setting for an enforcer?

1. Find which profile the enforcer is using (see above).

2. Policy Enforcer Configuration tab, Desktop Enforcers or File Server Enforcers link, locate the profile in the list at left, click the Settings tab, check the Heartbeat Frequency setting.
