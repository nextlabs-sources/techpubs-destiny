Replacing Policy Controller Certificates

The certificates necessary for Policy Controllers are distributed to Policy Con­trollers after their first registration with the Policy Server. In order to download the new version of these certificates, you must re-register with the Policy Con­troller. This must be done on all devices where Policy Controllers are installed. Follow these steps:

  1. On the host where the Policy Controller is installed, stop the Policy Server Enforcer Service using Control Panel > Administrative Tools > Services > Policy Server Enforcer Service.

  2. Locate the bundle.bin file at <installdir>\Policy Controller. Delete it.

  3. Locate the old certificates that were distributed on the default installa­tion at <installdir>Policy Controller\config\security. Delete the following files:

    4. ·        agent-keystore.jks

    ·        agent-truststore.jks

    ·        agent-secret-keystore.jceks

  4. Restart the Control Center Enforcer Service.

  5. Check that new agent files have been downloaded to the <install­dir>Policy Controller\config\security folder.
