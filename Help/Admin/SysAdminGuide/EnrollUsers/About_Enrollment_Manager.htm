About Enrollment Manager

You use the Enrollment Manager utility to enroll entities into the Network Infor­mation Directory, either directly from an LDAP directory or from an LDIF file. These entities include users and user groups, hosts and host groups, applica­tions, and SharePoint groups. The Enrollment Manager provides a number of enrollment management functions, described later in this chapter. It is not used in enrolling location sites or file shares; they have their own special utilities, for Windows and Linux.

Note: Enrollment Manager can only be used by persons who are defined as authorized users and assigned a System Administrator role. You define authorized users in Administrator, on the Users and Roles tabs; for details, see About Roles.

How Enrollment Connections are Established

The Enrollment Manager utility can be installed on any host in the network, as long as it can connect to the server where Control Center is running. Similarly, the AD or other LDAP server can be anywhere in the network, remote from the Control Center. This means there are two connections required: the Enrollment Manager’s connection to the Control Center Server, and the Control Center Server’s connection to the LDAP server or other enrollment domain.

The section describes the connection processes for single domain enrollment and a domain group enrollment. This section also introduces you to the various input files (connection, definition, filter, and configuration) which are used to configure these connections. More detailed information about the files used for enrollment is in the section About Enrollment Input Files.

Connection during an Enrollment

The connection process is as follows:

·        The connection between Enrollment Manager and the Control Center Server is defined at the command line, when you run the -enroll or -sync command. The -s (server) and -p (port) options (see Command Line Argu­ments, Enrollment Manager) point the Enrollment Manager to the Control Center Server.

·        For single domain enrollments, the command line provides the location of the connection file (the -a option), definition file (-d), and filter file (-f). The connection file establishes the connection between the Control Cen­ter and the LDAP server. The definition and filter files specify what infor­mation should be retrieved from the enrollment domain. For domain group enrollments, you use the -g command to provide the location of the configuration file, which references all connection, definition, and filter files.

·        The Control Center then uses the connection information to establish its connection to the LDAP server. The connection is defined in the first four lines of the connection file—server, port, login, and password. (See Connection Files  and Definition Files .)

·        In the case of a Domain Group enrollment, when the enrollment
(-enroll) of the first subdomain is complete, the Control Center locates the next subdomain listed in the configuration file, and continues enroll­ing until it completes the last subdomain.

About Secure LDAP Connection

You can configure LDAP connection(s) to use standard SSL protocols to establish secure communication between the Control Center and the LDAP Server. Config­uration steps include obtaining the LDAP server’s certificate and locating it on the Control Center host, inserting the certificate into the Control Center’s trust­store, and configuring settings in the connection file(s). (Obviously, in the cases of Domain Group enrollment, these steps would need to be performed for each subdomain). These steps are discussed in detail in How to Configure a Secure LDAP Connection.

Note: In the current release, LDAP and LDAPS are supported.

About Connections and AD Load Balancing

Note that if you are running your Active Directory on more than one host with a load balancing device sharing the AD authentication traffic, you must run your enrollment directly to one or the other hosts, not to the load sharing device. Specifically, this means the hostname and port specified in the connection file must refer to one of the AD hosts directly, not the load balancing appliance.

Before You Enroll

Before you enroll LDAP data, you must run the Control Center installation wiz­ard, and then start Control Center. Specifically, the Management Server compo­nent must be installed and running before you begin your enrollment.

LDAP Directory Requirements

Before you start, be sure any data to be enrolled from you LDAP directory meets the following requirements:

·        All entries must have an objectClass attribute.

·        All entries must belong to the corresponding object class, as shown below:

·        All entries must have an objectGUID attribute, which holds the static identifier of that entry in the source directory.

·        All user and host entries (also called “terminal” entries) must have an objectSid attribute. This attribute holds the system reference information for that entry—that is, the token or identifier that is used to identify the entity within the Windows system. This attribute is assigned to each user and host by the LDAP directory. If this attribute is not present, the entry will be ignored and a warning will be logged.

·        User entries must have a userPrincipalName attribute that serves as a unique, user-friendly name for this entry within the directory.

·        Host entries must have a dnsHostName attribute that serves as a unique, user-friendly name for this entry within the directory.

·        If you will be combining a Linux file server with a Windows AD system, you must configure your Samba server so that all users and groups map prop­erly between the Windows and Linux systems.

Requirements for Domain Group Enrollment

An additional requirement applies for Domain Group enrollment. Trust relation­ships between subdomains of a Domain Group must be manually configured to ensure that policy bundles are created with all subdomain information. This pro­cedure is discussed in Configuring Trusted Domains.

Using the Enrollment Manager

The Enrollment Manager is automatically installed on the Control Center host, in

\Program Files\NextLabs\PolicyServer\tools\enrollment

If you divided your Control Center components among several hosts in a distrib­uted installation, the Control Center host refers to the server where the Man­agement Server is installed. However, the Enrollment Manager may be installed anywhere in the network, as long as it can connect to the Control Center host.

To use the Enrollment Manager, open a command line window, navigate to this location, and use the command enrollMgr plus the specific command and argu­ments for the function you want. All commands are described in CLI Commands, Enrollment Manager, below; descriptions of all command line arguments are provided in Command Line Arguments, Enrollment Manager.

The -enroll and -update commands require a reference to a definition file and/or a connection file. Definition files provide information on how informa­tion elements in your source directory or file will map to the data structures inside Control Center’s data tables; for details, see Definition Files . Connection files provide information necessary to connect to the LDAP directory data source; for details, see Connection Files . For the Domain Group option, Configu­ration files list all subdomains in a Domain Group as well as the location of rele­vant connection, definition, and filter files. When you use the -enroll and -update commands, you only reference the configuration file. For details, see Configuration Files (Domain Groups Only).

The following table provides an overview of different arguments that can be supplied to establish connections and to reference the location of input files (connection, definition, filter, and configuration files).

Argument

Description

-t

Used with the -enroll and -update commands to specify the type of enrollment. Valid values are:

  • DIR: enrolling from an LDAP server

  • PORTAL: enrolling SharePoint group definitions

  • LDIF: enrolling from an LDIF file

  • DOMAINGROUP: enrolling multiple subdomains in a single enrollment (which can include DIR, PORTAL, and LDIF type enrollments)

Note that these values are case sensitive, and must be typed in block caps as shown.

-n <domain_name> or <enrollment_name>

The name of the enrollment. Each enrollment needs a unique name, so you can identify it in various management functions, such as confirming, synchronizing, and updating. For User and Host enrollments, this value must be the name of the domain from which you are enrolling. Accordingly, you cannot perform more than one enrollment from a single domain; rather, you can perform an enrollment, then later update it (using the same name) to incorporate any changes.

For SharePoint or Application enrollments, you can use any name you like here—it does not need to be the domain name. This allows you to perform multiple enrollments from the same SharePoint server, as long as you give them unique names.

A Domain Group also receives its own domain name, as does each subdomain in the Domain Group (subdomain enrollment names are defined in the Configuration file).

-g <domaingroup file>

The location of the Domain Group configuration file. For more information on defining the Configuration file, see Configuration Files (Domain Groups Only)

-s <server>

The name or URL of the host where your Control Center is running. If you have a distributed Control Center installation where Enrollment Manager is installed on another host, this server should be the host where the Enrollment Manager is installed. If this argument is not present, the default value localhost will be used.

-p <port>

The port of the Control Center host, which is 8443 unless it was explicitly changed. This argument is optional; if it is not present, the default value 8443 will be used.

-u <username>

User name of an Control Center user with administrator privileges. We recommend that, rather than an actual live user, you create a dummy username and password, dedicated only to enrollment functions. This prevents the problems that can arise when a live user changes a username or password.

-w <passwd>

Password for the Control Center administrative user specified by -u. (This should be a dummy user dedicated for enrollment functions only, as described above.)

-a <connection_file>

The name of the directory connection file, which is required for enrolling or updating with an LDAP server and for enrolling portal sites. You can create a connection file by copying one of several provided template or default files, depending on which kind of entity you are enrolling; for a detailed explanation, see Connection Files .

-d <definitionfile>

The name of the definition file, which is required for all enrollments and updates.You can create a definition file by copying one of several provided template or default files, depending on which kind of entity you are enrolling; for a detailed explanation, see Definition Files .

-f <filterfile>

The name of the LDAP filter file, if you are using one. Optional in all cases; see Filter Files.

-v

Verbose option; may be used with any command. When present, if an error condition occurs the command line window will display a more detailed description of the circumstances of the error. If not present, only a short summary of any errors will display.
