About Configuration Files

You perform all configuration by editing various Control Center configuration files directly, and saving your changes.

The platform uses XML files that contain configuration settings you can view or modify in order to tune your system’s performance, scale the system by adding or removing modules, or address various administration tasks and issues. The primary configuration files are:

·        configuration.xml: The main configuration file for Control Center. This file contains settings for Control Center operation, such as heartbeat fre­quency and other settings that are required for the various Control Center software components to communicate.

·        server.xml: The configuration file for the Tomcat application server, which is part of the Control Center platform. You only need to edit this file if you install additional instances of the Tomcat server.

·        ddac_config.xml: The configuration file for the DAC Bridge Server. You only need to change this file if your implementation includes DAC policies to be deployed to Active Directory.

Note: For more information on DAC policies, consult the Policy Studio User’s Guide.

These files begin with a set of initial configuration settings. In order to modify those initial settings, or to access settings that are not included in the installa­tion wizards, you can edit the configuration files directly. Each feature is associ­ated with a section in the file, and focuses on the functions of each information element (marked by its XML tag) in that section. An element may or may not have one or more Properties sections within it, which control aspects of the cor­responding feature’s operation. To change any aspect of your configuration:

  1. Open the file with a text editor.

  2. Find the setting you are interested in by searching for its tag.

  3. Change the value between the open and close tags.

  4. Close the file, saving your changes.

The Control Center Configuration File

The main Control Center configuration file, configuration.xml, is placed on the host machine when you first install the Control Center. It controls the behavior of all Control Center components throughout your system, even if they are installed on different hosts. During installation, the file is placed in

<InstallDir>\server\configuration\configuration.xml

You can use this file to change the configuration of the following:

·        User repository

·        Authentication

·        Control Center components

·        Data access

·        Information Network Directory

The Tomcat Application Server Configuration File

Control Center uses the Tomcat open-source application server to provide a platform for the Control Center server components. If you install Control Center components in a distributed architecture rather than on a single host, Tomcat is installed on each of the host machines.

Every installation of Tomcat has its own configuration file, called server.xml. This file informs a given instance of Tomcat about which applications it will be serving on that machine. During installation, the file is placed in

<InstallDir>\server\configuration\server.xml

The Control Center implementation of the Tomcat configuration file contains many standard Tomcat settings, which you can modify to tune your system. For example, you can change the default number of threads and various time-outs. More specifically to Control Center, the file contains <Service> sections that pro­vide settings for two services, DCC-Core and DCC-Apps. You must not delete either of these sections, but you can modify some of their properties. For more details, see Tomcat Settings.

The DAC Configuration File

NextLabs Dynamic Access Control (DAC) policies allow you to integrate NextLabs policy enforcement with Windows Server 2012 DAC rules. This integration is enabled by the NextLabs DAC Bridge Server, which is installed automatically with NextLabs Control Center.

Before you can deploy DAC policies, you must configure DAC Bridge Server set­tings in the DAC Configuration file (ddac_config.xml). This file stores connec­tion information that allows the DAC Bridge Server to locate and connect with the target AD Server(s). This file also defines attribute-to-claim and attribute-to-property mapping, which the DAC Bridge Server uses to convert NextLabs pol­icies to Microsoft CARs.

Note: Because this configuration is related to policies, all procedures for the DAC Bridge Server (including the ddac_config.xml file) are discussed in the Policy Studio User’s Guide.
