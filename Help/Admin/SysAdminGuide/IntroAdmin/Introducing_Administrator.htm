Introducing Administrator

Control Center administrators use the Administrator application for routine sys­tem management tasks, including:

·        Monitoring the operational status of Control Center processes, File Server Enforcers, Desktop Enforcers, and SharePoint Enforcers

·        Monitoring statistical summaries of the policy enforcement activity of the system

·        Defining and assigning the configuration profiles that control several aspects of enforcer behavior

·        Monitoring the status of policies deployed to enforcers—that is, whether they are up-to-date, or have been revised but are awaiting deployment

·        Defining and managing administrative users authorized to access the Control Center tools

Launching Administrator

Because Administrator runs as a web application, it can be launched from any web browser that can connect to the server where the Administrator Server is running.

Note: If you are using Firefox as your Web browser, you cannot access Admin­istrator unless you enable TLS 1.0 (under Tools, Options, Security). This is not required with Internet Explorer.

To launch Administrator:

  1. In your Web browser, enter the URL where Administrator is installed at your organization. Typically, the URL is in the following form:

    2. https://<hostname>/administrator

    The Login dialog appears.

  2. Type your user name and password. If you have not yet been assigned a user name and password, see your system administrator.

    3. You can also use the built-in super administrator, Administrator, along with the password set during Control Center installation. Typically, only a Control Center system administrator knows this password and can use this account.

  3. Click Login. The Administrator main window appears.

To exit Administrator, click logout in the upper right of the window. Alterna­tively, simply close the browser.

Release Version

The release version of the Administrator web application is displayed under the logo in the upper right corner, as shown above. The number in parentheses is the build number for that release. This build number reflects not only Administrator, but also the version of the Control Center—that is, the entire Control Center platform—you are using.

Changing Passwords

The Change Password link at the upper right of the screen can be used to change the password of whatever user is logged in. If you are logged in as the super user, Administrator, you can change the super user password. Note that this password is also used for system utilities, such as Enrollment Manager and Property Manager.

Administrator’s Tab Structure

The Administrator work space is organized into several screens that you navigate using the following main tabs:

·        Status

·        Users and Roles

·        Policy Enforcer Configuration

Below each of these tabs are sub-tabs, which you click to drill down to addi­tional screens. The rest of this chapter describes each of these screens and how to use them in routine administrative tasks.
