Managing Control Center

All the server and database components of Control Center are designed to start once and run continuously as a Windows service. Users should note that the name of this service does not reflect the individual server components. In fact, although you may have several or even all of the Control Center server compo­nents installed on the same host, they all appear in the Windows Services man­ager as a single service, called Control Center Policy Server. Similarly, no matter how you distribute components across several hosts, each host displays only one Control Center Policy Server service in the Windows Services list, representing one or more server components.

However, the embedded databases—the Administrator database and the net­work directory—do appear as separate services in the list, and may be restarted manually if necessary.

Starting and Stopping

Whether running on a single host, on distributed hosts, or in a failover cluster, the Control Center Policy Server service must be manually started after the ini­tial installation. By default, it is not set to restart automatically if it fails, but like any service, you can change that if you wish, by changing the Startup Type from Manual to Automatic.

You can use the Status tab in Administrator (see The Status Tab) to monitor whether the individual components are running normally. If one fails and you need to restart it, you can do so by restarting the Policy Server service, regard­less of whether that service represents more than one Control Center compo­nent.

Similarly, if you ever need to stop any server process, you must stop the service even though this stops all Control Center components running on that host.

Changing Memory Settings

You might encounter out-of-memory errors during data-intensive operations, such as enrolling large amounts of data, generating a report with more than 50000 records, or synchronizing a large number of activity logs. If this occurs, change the JVM memory settings in the Windows Registry, using the following procedure:

  1. Open the Registry Editor. You can do this by clicking the Windows Start button, then in the search box, type regedit, and press Enter.

  2. In the Registry Editor, navigate to the following folder:

    3. HKEY_LOCAL_MACHINE\SOFTWARE\Wow6432Node\Apache Software Foundation\Procrun 2.0\CompliantEnterpriseServer\Parame­ters\Java

  3. Open Options and look for the following settings. Increase the values as needed.

-Xms256M
-Xmx1024M
-XX:MaxPermSize=256M

Depending on the load on Control Center, try one of the following settings:
-Xmx2048M or -Xmx3072M or -Xmx4096M.
